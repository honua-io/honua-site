Honua vs. Esri, GeoServer & QGIS Server

← Back to the proof hub

The other proof-hub pages go deep on a single axis — the Compatibility Matrix on Esri-client coverage, Benchmarks on throughput and latency. This page is the side-by-side: how Honua lines up against the three servers an evaluator is most likely already running — Esri ArcGIS Server, GeoServer, and QGIS Server — protocol by protocol.

How to read this page. Cells marked sourced are quoted from an open, re-runnable artifact already cited elsewhere in this hub — the geobench headline snapshot and the honua-esri-compat matrices. Cells marked positioning are architectural facts about each product's design (e.g. "Java/JVM", "per-named-user licensing") rather than a measured number — they are not a benchmark result and are not presented as one. No competitor list prices or made-up performance numbers appear anywhere on this page.

The honest summary

  • vs. GeoServer: same open-source freedom, but Honua led every comparable cell in the benchmark snapshot and adds a native Esri GeoServices surface and OpenTelemetry. Sourced.
  • vs. QGIS Server: QGIS Server has no GeoServices REST surface and no STAC/OData; Honua covers the same OGC surface plus Esri clients on one server. Positioning.
  • vs. ArcGIS Server: Esri clients keep working against Honua (open compatibility evidence), without the per-named-user tax or runtime license server — at the cost of catalogued-operation gaps that are published honestly on the Compatibility Matrix and Migration Evidence pages. No first-party performance comparison is claimed.
