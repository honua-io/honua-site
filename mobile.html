Field Toolkit

Field work on the same runtime as everything else.

A MAUI-first field toolkit with GeoPackage-backed offline storage, background sync, and form workflows. One codebase across iOS, Android, and Windows. Same data model, same auth, same operations as the central platform.

What it covers

Offline storage, background sync, guided collection.

Offline

GeoPackage storage with queued edits.

Maps, features, and edits persist on the device. Field teams keep working through coverage gaps; edits flush when the network returns.

Sync

Background upload with conflict handling.

gRPC-friendly transport, sync cursors, and conflict resolution so a week-long offline stretch doesn't mean a week of manual reconciliation.

Forms

Validation, calculated fields, workflow rules.

Collection forms with real field logic — required photos, conditional sections, derived values — not a generic data-entry screen with asterisks.

Same platform, all the way to the device

Field tools that share the runtime, not copy it.

Shared data model

The schema your analysts use is the schema your field app uses.

Collections, field definitions, and validation rules are defined once on Honua and consumed by MAUI clients. No separate mobile data model to keep in sync.

Shared operations

The same auth, traces, and rollout pipeline.

Mobile requests flow through the same gRPC runtime and surface in the same OpenTelemetry traces. Your platform team doesn't operate a second stack for field work.

What ships

Toolkit components.

Component Purpose Notes
Transport + auth Connect field apps to Honua services Uses the same .NET SDK as backend services.
Offline engine GeoPackage storage for maps and edits Queues edits, flushes on reconnect.
Forms + workflows Structured collection with validation Calculated fields, conditional logic, required media.
MAUI integration One codebase across iOS, Android, Windows XAML UI, platform APIs via MAUI abstractions.
