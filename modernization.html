How teams move

Import. Parity-check. Shift new work.

Three steps, in order. Each one is validated before you move to the next.

1 · Import

Pull your current services into Honua.

Service importers clone definitions, metadata, and runtime configuration from your existing ArcGIS or GeoServer estate. Nothing gets recreated by hand.

2 · Parity-check

Compare output side by side.

Reconciliation tooling flags mismatches in data, rendering, response shape, and workflow behavior before a single user sees anything change.

3 · Shift

Move new work to gRPC, SDKs, and agents.

Legacy clients stay on the compatibility surface. New apps go to typed contracts, offline-first SDKs, and agent workflows. Both run on the same deployment.

Migration automation

What the toolkit does for you.

Three tools cover the parts of a migration that are genuinely repetitive. The judgement calls stay with your team.

Service importers

ArcGIS and GeoServer → Honua in one pass.

Reads service definitions, publishes them on Honua, and preserves URLs where possible so clients don't need reconfiguration.

Client code migration

Rewrite ArcGIS JS calls to Honua SDK.

Scans JavaScript codebases, identifies the repeatable ArcGIS client patterns, and replaces them with equivalent SDK calls. Leaves the non-trivial ones for you.

Parity checks

Catch drift before rollout.

Runs your existing services against Honua and compares features, tiles, rendered maps, and response shapes. Surfaces the diffs as an actionable list.

Failure modes

The usual migration mistakes — and how this avoids them.

Old approach Where it breaks Honua approach
Rewrite apps before the runtime is validated Service behavior drifts; teams find out in production Parity-check the runtime first. Move apps once it passes.
Recreate services by hand Scope grows every week; translation errors compound Importers do the mechanical work. Humans review the diffs.
All-or-nothing cutover One dependency fails, everything rolls back Legacy and modern clients run against the same service. Cut over piece by piece.
The compatibility surface buys you time. The new surface earns its keep by shipping better apps.
