Pricing and Licensing

Scales with infrastructure, not headcount.

Community is free. Commercial is annual and per-node. No per-user fees. Protocols, SDKs, mobile access, and agent workflows are never metered or gated. Standard cloud deployments should usually start with self-directed evaluation and annual packaging, not a custom deal.

Commercial pricing is quote-based today. Standard cloud deployments should usually move into annual per-node commercial terms directly. Assisted migration assessments and fixed-scope pilots are reserved for higher-complexity migrations, not the default path. Three cost components: software subscription, support tier, and optional professional services. Customer cloud infrastructure spend is separate.

Commercial Paths

Three ways commercial work gets structured.

Keep the default path simple. Escalate only when the migration or procurement shape actually requires it.

Standard Cloud Path

Self-directed evaluation, then annual per-node terms.

Best fit for straightforward cloud deployments that can evaluate with docs, Docker, SDKs, and a normal procurement path. This is the default commercial motion.

Complex Migration Path

Assessment and fixed-scope pilot.

Use this when client continuity, data movement, auth, field workflows, or procurement complexity needs a human-guided plan before the annual software agreement.

Direct Exception Path

Reserved for deployment or procurement edge cases.

Air-gapped, on-prem, regulated, or otherwise non-standard environments may need a direct exception structure. Those should stay narrow and documented.

Private

AI DevOps operator

Support Model

Agent-first support, controlled write paths.

Honua support is designed to minimize overhead while keeping remediation safe and auditable.

Observe

Start with customer-provided artifacts and read-only telemetry. Most tickets should begin here.

Guided fix

The default paid support path is commands, diffs, rollback steps, and validation checks the customer can run.

Approved write path

Premium support can include GitOps or control-plane mediated changes with explicit approval and review.

Break-glass

Direct changes in customer environments are narrow exceptions with named approval, time limits, and audit evidence.

Next Step

Start self-directed. Escalate only if the migration is complex.

The fastest path is docs, Docker, and one SDK. If the rollout or migration is more involved, use the assessment path instead of improvising the project in email.
