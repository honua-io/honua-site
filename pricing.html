Open Core

Scales with infrastructure, not headcount.

Community is free. Commercial is annual and per-node. No per-user fees. Protocols, SDKs, mobile, and automation surfaces are never metered or gated.

Commercial Paths

Three ways commercial work gets structured.

Standard Cloud Path

Self-directed evaluation, then annual per-node terms.

Best fit for straightforward cloud deployments that can evaluate with docs, Docker, SDKs, and a normal procurement path. This is the default path.

Complex Migration Path

Assessment and fixed-scope pilot.

Use this when client continuity, data movement, auth, field workflows, or procurement complexity needs a human-guided plan before the annual software agreement.

Direct Exception Path

Reserved for deployment or procurement edge cases.

Air-gapped, on-prem, regulated, or otherwise non-standard environments may need a direct exception structure. These are handled case by case.

Commercial Packaging

Software, support, and services are separate lines.

Software Subscription

The platform license covers the product surface.

The commercial software subscription covers the server runtime, protocol APIs, SDKs, mobile toolkit, admin surface, public MCP package, data services, and cloud-native formats.

Support + Services

Support tiers are separate from delivery work.

Support SKUs cover response times and support scope. Assessments, pilots, migrations, and scoped delivery work are professional services, not hidden inside the software price. GitOps change management is part of the product, not a services add-on.
