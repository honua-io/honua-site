// pricing & licensing

Priced like infrastructure.
Because it is infrastructure.

Honua Server is ELv2 — source-available, free to read, and free to run yourself. Every client SDK is Apache 2.0. Commercial tiers are licensed per node, annually — counted by capacity, not processor cores, with unlimited users. We publish the model so you can verify it is predictable before you ever talk to us; we are in early access, so the numbers are set in a short conversation, shaped with our first customers.

// the model

What the model guarantees.

You buy a capacity band that covers your everyday fleet; your variable cost is your own cloud infrastructure, which you already know how to price.

Cost scales with workload, not adoption
Adding your thousandth user costs nothing. No named users, no credits, no per-core arithmetic.
One flat annual number
Your invoice does not vary within the term. Autoscaling inside your band is free, including short bursts above it.
Growth is forward-only
Sustainably outgrow your band and you pay the pro-rated difference to the next one from that day forward — never backward. No true-ups, ever.
Size down at renewal
If your footprint shrinks, so does your price. We advertise this because almost nobody else offers it.
Emergency surge always works
Declare an emergency and the capacity ceiling lifts entirely — scale twenty-fold during the storm without a licensing conversation. Surge coverage with a response SLA is available for emergency-mission organizations.
There are no audits
The license enforces its own capacity band, and you see the same meter we do. The dashboard is the audit.
Non-production is free
Dev, test, staging, and cold-standby DR capacity don't count toward your band.
Topology-neutral
VMs, Kubernetes, or serverless — licensed by concurrent serving capacity, so re-architecting never reprices you.
The license never touches a request
It sizes your fleet, nothing else. No per-request pricing, no per-GB overages, no metered invoices, no throttling.

// what's in each tier

Community, Pro, Enterprise — by capability.

This is the edition gating that ships in the product today, drawn from the in-product feature catalog — not a marketing wishlist. Pricing is still being shaped with early customers, and the boundaries can still move as the tiers are finalized. Each tier includes everything in the tier to its left.

Community · free · ELv2

Source-available — free to read, serve, and run yourself.

  • All OGC & GeoServices read, query, tiles, and metadata surfaces
  • File import — GeoJSON, Shapefile, GeoPackage
  • ArcGIS Portal token issuance + read-only Portal sharing facade (Esri-client interop)
  • Smart geometry-aware style defaults
  • Temporal query filtering + extent discovery
  • COG import / export
Pro · pricing TBD

Everything in Community, plus editing, operational, analytics, and migration tooling.

  • Multi-user editing via FeatureServer — applyEdits / add / update / delete
  • Geofence alerts (enter/exit) + evaluation engine · webhook delivery
  • Forward / reverse geocoding + provider failover
  • Output caching + Redis distributed cache (multi-node)
  • Real-time feature streams — WebSocket / SSE with replay cursors
  • High-DPI, large-dimension, and rich-overlay static maps
  • Auto-cartographic styling
  • COG serving from S3 / Azure + temporal raster mosaic
  • Spatial analytics — clustering, spatial join, buffer aggregate, density, sun/shadow, slice, line-of-sight, viewshed
  • Temporal animation — date-bin histograms, time-series tiles, animation API
  • PDF print output + full layout templates
Enterprise · pricing TBD

Everything in Pro, plus the enterpriseready.io baseline and heavy-integration features.

  • SSO — OIDC multi-provider (Azure AD, Google, generic) + claim-to-role mapping
  • Dwell + attribute-threshold alert triggers
  • Alert channels — email, Slack, Microsoft Teams, AWS SNS, Azure Event Grid, digest
  • Batch geocoding
  • Source migration import — ArcGIS REST (GeoServices) + GeoServer
  • Esri-style branch versioning — named gdb versions, reconcile/post, VersionManagementServer (PostgreSQL)

// procurement

Buy through your cloud marketplace.

Standard cloud deployments transact through AWS Marketplace (Azure following) as flat annual contracts — the purchase draws down your existing committed cloud spend and rides the marketplace standard contract. For public-sector buyers, that is a PO against a vehicle you already have, not a six-month legal negotiation. Negotiated terms, multi-year agreements, and larger capacity run as marketplace private offers.
