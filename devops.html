Manifest example

service:
  name: parcels
  source: postgis://operations/parcels
  protocols: [geoservices, ogcapi, grpc]
  style: parcels-default
  permissions:
    readers: [planning, public-map]

GitOps publishing

Services, styles, and permissions — reviewed like code.

Declarative

YAML manifests, not UI clicks.

Publish a feature service, a style, or a permission change by committing a file. Every setting is in the repo.

Reviewable

Pull requests, CI, environment promotion.

Changes flow through the same gates as the rest of your stack. No production console with a big save button.

Reversible

Roll back with git revert.

The repo is the source of truth. Every change has an author, a timestamp, and an approver. Recovery is a known flow.

OpenTelemetry

Stop guessing why a spatial request is slow.

Honua exposes traces so teams can follow a request from ingress through the runtime and into the database. That makes performance tuning and incident response concrete instead of guesswork.

Request visibility

One trace from client, through the runtime, into PostGIS.

Works with Grafana, Datadog, Honeycomb, or any OTel-compatible backend. No GIS-specific dashboards to build and maintain.

Shared context

Platform and GIS teams see the same thing.

When a spatial request is slow, both sides look at the same trace and agree on where the time went. No handoff, no reproduction steps required.
