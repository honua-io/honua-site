Platform

A full-stack geospatial platform, not just a feature server.

One platform replaces your GIS server, mapping stack, field tools, and SDK layer. Self-hosted, cloud-native, and ready for migration today.

Core Idea

Replace the runtime. Then replace the rest of the stack.

Move GIS services onto Honua without breaking existing clients. Then bring in mobile, analytics, raster services, and data services on the same platform instead of stitching together five different products.

Compatibility

Keep existing clients. Fix the operating model.

ArcGIS Pro, QGIS, GeoServer-facing clients, Excel, Power BI, and existing web maps keep working on day one. Import live services, migrate apps with SDK tooling, and move new work onto gRPC and MCP at your own pace.

Open Standard

An open geospatial gRPC protocol, built here first.

GIS has been stuck on REST/JSON and XML for two decades. Honua is building a typed binary protocol — ~60–70% smaller payloads, up to 5× faster parsing — and releasing it as an open specification targeting OGC working-group submission.

  • Proto definitions and spec published under an open license.
  • Targeting OGC working-group submission as a community standard.
  • No vendor lock-in: any server or client can implement the spec.

Cloud-Native Deployment

GIS that deploys like the rest of your infrastructure.

Legacy GIS servers require bespoke Windows installs, proprietary clustering, and dedicated ops teams. Honua ships as a single Linux container image that deploys with the same tools your platform team already uses — Docker, Kubernetes, Terraform, Helm. Stateless runtime. Standard ingress. PostGIS for storage. No special infrastructure.

Serverless

The first serverless-capable GIS server.

Honua runs on AWS Lambda, Azure Functions, and other serverless platforms — something no other GIS server can do. The runtime is fully stateless: cold starts are fast, scaling is automatic, and you pay only for the requests you serve. No idle servers, no capacity planning, no dedicated GIS infrastructure.

  • AWS Lambda with API Gateway and RDS PostgreSQL.
  • Azure Functions with Azure Database for PostgreSQL.
  • Same protocol surface as container deployments — nothing is removed.

Containers

Docker, Kubernetes, ECS, AKS — the standard paths.

For persistent workloads, Honua ships as a single Linux container image. Docker Compose for local dev, Helm charts for Kubernetes, Terraform modules for ECS/Fargate and AKS/Container Apps. OpenTelemetry tracing, Prometheus metrics, and Grafana dashboards are built in.

  • Production-ready Helm charts with autoscaling and TLS.
  • Terraform modules for AWS (ECS/Fargate, RDS, ElastiCache) and Azure (AKS, Container Apps).
  • OpenTelemetry + Prometheus + Grafana in every deployment target.

One cloud-native runtime behind standard ingress, backed by PostGIS, optional Redis, object storage, and observability.

Self-hosted is the default — on containers or serverless. Start with docs and deployment templates, or reach out when you need hands-on migration help.

What Ships

What you get in the platform today.

Runtime

Multi-protocol server

FeatureServer, MapServer, ImageServer, Geometry Service, OGC API Features/Tiles/Maps, OData v4, vector tiles, WMS, WMTS, gRPC, MCP, and WFS 2.0 with transactions from one PostGIS-backed process.

Admin

Admin APIs and Blazor UI

Publish services, manage connections and layers, configure styles, import data, and govern the runtime through REST admin APIs and a Blazor WebAssembly control center.

SDKs

JavaScript, .NET, Python

Typed clients, gRPC support, admin automation, geocoding/routing access, migration tooling, and an MCP server package. All Apache 2.0.

Mobile

MAUI field toolkit

Offline GeoPackage sync, dynamic forms, background upload, gRPC transport, and AR-assisted and sensor-aware field collection.
