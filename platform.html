Platform Overview

One runtime for client continuity, modern application delivery, and AI-ready spatial work.

Honua brings the parts teams usually manage separately into one platform: standards and compatibility for current clients, gRPC and SDKs for new apps, MCP for AI workflows, and Git-friendly operations for production rollouts.

One Deployment

Current clients and new workloads share the same backend.

Honua is designed so compatibility is not a separate product and gRPC APIs are not a separate sidecar. Everything runs against the same data layer, the same admin surface, and the same deployment footprint.

Workload Interface Why teams use it
ArcGIS Pro, legacy web apps, Esri integrations GeoServices REST, MapServer, ImageServer, Geometry Service Preserve current client behavior while the platform changes underneath.
QGIS and standards-based clients WFS 2.0, OGC API, WMS, WMTS Keep open-standard interoperability in place.
Application backends, services, mobile sync gRPC and SDKs Lower overhead, clearer contracts, and faster integration work.
AI assistants and governed automation MCP Give tools a discoverable, structured way to work with spatial data.
Analytics and business tooling OData and standards APIs Bring spatial data into downstream systems without custom export jobs.
One runtime means one data model, one operational surface, and one place to apply rollout, security, and observability practices.

Operational Footprint

Run it where your platform team already works.

Honua ships for standard cloud paths. Run it in containers for always-on traffic or use serverless footprints for event-driven workloads without standing up separate GIS infrastructure.

Serverless

Scale spatial processing without paying for idle capacity.

Use AWS Lambda or Azure Functions for request-driven spatial jobs, background tasks, and bursty workloads while keeping the same core interfaces.

Containers

Deploy through Docker, Kubernetes, ECS, AKS, or the path you already own.

Package the runtime as standard infrastructure, integrate it with managed PostGIS, and feed traces and metrics into the dashboards the rest of your services use.

A single runtime can serve compatibility traffic, application APIs, AI tooling, and operational telemetry from the same deployment.
