Platform overview

One runtime. Five capabilities. Shared user classes.

Honua is a single binary that serves documented compatibility clients, gRPC applications, Preview AI agent surfaces, and Beta field workflows from the same deployment against the same data. This page is the architectural tour.

The five capabilities

What the runtime does.

Each capability is first-class. Each has its own page with code, examples, and depth.

01 · Talk to your data

Natural-language spatial access over MCP.

Claude, GPT, Cursor, or your own agent discovers collections, inspects schemas, and runs typed queries. For analysts, teammates, and assistants alike.

See AI agents

02 · SDKs for apps

Typed clients in JS, Python, .NET, and Beta MAUI.

Preview client surfaces span web, backend, desktop, and pilot field workflows. gRPC is the shared protocol under the hood.

See the SDKs

03 · Cloud-native runtime

gRPC + standards in one binary.

Typed Protobuf contracts for new apps, standards APIs for existing ones. Both served from the same process.

See the runtime

04 · Compatibility

Standards with published evidence.

137/137 OGC API Features plus documented WFS, WMS, WMTS, and GeoServices REST coverage with partial-parity caveats.

See compatibility

05 · Operate like software

GitOps, OpenTelemetry, AI DevOps.

Services in Git, rollouts in PRs, traces through PostGIS. Six AWS and Azure deployment paths are tracked for Preview validation.

See operations

+ Field toolkit

Beta MAUI field toolkit for iOS, Android, Windows.

GeoPackage storage, sync, and form workflows remain Beta or pilot-scoped until field proof assets are linked.

See the field toolkit

One runtime, many surfaces

Documented interfaces share the same process.

Compatibility clients, new apps, Preview AI agent surfaces, and analytics tools use the same runtime boundary. Auth, permissions, and telemetry apply through the documented surfaces.

Audience Interface What they get
ArcGIS Pro · Esri web apps · ArcGIS JS API GeoServices REST, MapServer, ImageServer, Geometry Service Documented compatibility paths, with parity gaps visible before rollout.
QGIS · OpenLayers · Leaflet · Mapbox GL OGC API, WFS, WMS, WMTS, MVT Standards-compliant access. Point the layer at Honua.
New apps, backends, Beta field sync gRPC + SDKs (JS / Python / .NET / Beta MAUI) Typed contracts, lower overhead, generated clients.
AI assistants and automation MCP agent tools Discoverable schemas and typed spatial operations.
Analytics, BI, pipelines OData + standards APIs Spatial data into downstream systems without export jobs.
One runtime means one data model, one auth boundary, and one place to apply rollout, security, and observability.

Deploy targets

Run it where your platform team already runs everything else.

Six deployment paths on AWS and Azure are tracked for Preview validation. Terraform module evidence remains owned outside this site in honua-terraform.

AWS

ECS / Fargate · Lambda · EKS

Managed containers, serverless for event-driven work, and Kubernetes when your platform team already runs a cluster. Pick the one that matches the workload.

Azure

Container Apps · Functions · AKS

Same shape on Azure — container-first, serverless where it fits, or full AKS for Kubernetes-native platforms. Same manifests, same traces.

One runtime serves compatibility traffic, application APIs, Preview AI agent tooling, and Beta field sync from the same deployment boundary.
