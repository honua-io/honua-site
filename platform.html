Platform overview

One runtime. Five capabilities. Every user class.

Honua is a single binary that serves compatibility clients, gRPC applications, AI agents, and field tools — from the same deployment, against the same data, on the compute your team already runs. This page is the architectural tour.

The five capabilities

What the runtime does.

Each capability is first-class. Each has its own page with code, examples, and depth.

01 · Talk to your data

Natural-language spatial access over MCP.

Claude, GPT, Cursor, or your own agent discovers collections, inspects schemas, and runs typed queries. For analysts, teammates, and assistants alike.

See AI agents

02 · SDKs for everything

Typed clients in JS, Python, .NET, and MAUI.

One client surface across web, backend, desktop, and field. gRPC under the hood; offline-first sync where you need it.

See the SDKs

03 · Cloud-native runtime

gRPC + standards in one binary.

Typed Protobuf contracts for new apps, standards APIs for existing ones. Both served from the same process.

See the runtime

04 · Compatibility

Standards everywhere, conformance published.

137/137 OGC API Features, WFS, WMS, WMTS, GeoServices REST. Your ArcGIS Pro and QGIS clients keep working.

See compatibility

05 · Operate like software

GitOps, OpenTelemetry, AI DevOps.

Services in Git, rollouts in PRs, traces through PostGIS. Six validated deploy targets on AWS and Azure.

See operations

+ Field toolkit

MAUI offline for iOS, Android, Windows.

GeoPackage storage, background sync, form workflows. Same runtime, same data model, all the way to the device.

One runtime, many surfaces

Every interface ships in the same process.

Current clients, new apps, AI agents, and analytics tools all read and write the same data. Auth, permissions, and telemetry apply uniformly.

Audience Interface What they get
ArcGIS Pro · Esri web apps · ArcGIS JS API GeoServices REST, MapServer, ImageServer, Geometry Service Existing clients keep working. No porting project.
QGIS · OpenLayers · Leaflet · Mapbox GL OGC API, WFS, WMS, WMTS, MVT Standards-compliant access. Point the layer at Honua.
New apps, backends, mobile sync gRPC + SDKs (JS / Python / .NET / MAUI) Typed contracts, lower overhead, generated clients.
AI assistants and automation MCP agent tools Discoverable schemas and typed spatial operations.
Analytics, BI, pipelines OData + standards APIs Spatial data into downstream systems without export jobs.
One runtime means one data model, one auth boundary, and one place to apply rollout, security, and observability.

Deploy targets

Run it where your platform team already runs everything else.

Six deploy targets on AWS and Azure, validated end-to-end in CI. Terraform modules for every one live in honua-terraform.

AWS

ECS / Fargate · Lambda · EKS

Managed containers, serverless for event-driven work, and Kubernetes when your platform team already runs a cluster. Pick the one that matches the workload.

Azure

Container Apps · Functions · AKS

Same shape on Azure — container-first, serverless where it fits, or full AKS for Kubernetes-native platforms. Same manifests, same traces.
