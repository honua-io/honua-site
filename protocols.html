OGC submission path

The gRPC spec is going through the standards body, not held private.

The typed, high-performance spatial APIs that power the runtime are being proposed through the OGC process so anyone can adopt the wire format — whether they run Honua or not.

Protocol definitions

Open Protobuf schemas for feature, map, tile, and form services.

Published at honua-io/geospatial-grpc. Reuse them in your own services, or generate a client in any language with protoc.

Agent protocol

An open MCP schema for geospatial agent workflows.

Published at honua-io/geospatial-mcp. Gives Claude, GPT, Cursor, or your own agent a common way to work with spatial data safely.

API surface

Standards, compatibility, and modern APIs — one deployment.

Everything below ships in the same binary. You can keep a WFS client and a gRPC client pointed at the same service at the same time.

Surface What it covers When to reach for it
Standards APIs OGC API Features, Tiles, Maps; WFS, WMS, WMTS Existing desktop, web map, and partner clients.
Compatibility APIs GeoServices REST, raster services, geometry services Systems built against ArcGIS Server or GeoServer.
Modern APIs gRPC, SDKs (JS / Python / .NET / MAUI), MCP New apps, automation, field collection, and AI agents.
Pick the surface that fits the job. The same data, auth, and operations back all three.
