OGC Submission

Published standards evidence for the runtime you can use today.

Honua is building its spatial runtime in the open. This page presents the published conformance results and the open standards path for the gRPC specification work.

Why Publish It

Customers should be able to inspect both the evidence and the direction.

Standards work matters when teams want performance and interoperability at the same time. Publishing conformance counts and the submission path makes the runtime easier to evaluate than a closed performance claim.

Conformance

Standards compatibility is measured, not described vaguely.

Honua publishes exact conformance counts for the standards surface so teams can see what is implemented and how the runtime was validated.

Submission Path

The gRPC work is being shaped as open standards work, not a private shortcut.

The goal is to make typed, high-performance spatial APIs easier to trust, evaluate, and adopt over the long term.
