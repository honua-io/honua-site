Protocols

One runtime. Three access lanes.

Honua brings GeoServices REST, OGC API, OData, gRPC, MCP, and cloud-native tiles together in one runtime. Legacy GIS clients can keep working while new apps, mobile teams, analysts, and agents move onto better interfaces.

Differentiated Rails

The protocols that move Honua beyond legacy GIS.

gRPC

Application and mobile performance rail

gRPC gives applications and mobile clients a typed binary rail for higher-throughput requests, service-to-service calls, and richer field workflows than legacy REST and XML patterns can support.

  • Strong fit for typed SDKs, field apps, and service integrations.
  • Honua mobile protocol work targets roughly 60-70% lower bandwidth and 5x faster loading in benchmark/spec work.
  • The protocol direction is aimed at OGC working-group submission, not a private wire format moat.

MCP

Agent-native geospatial access rail

MCP lets coding agents, copilots, and analyst assistants discover services, inspect schemas, and run filtered queries through a supported interface instead of brittle scraping and custom wrappers.

  • Discovery and query workflows for AI assistants and coding agents.
  • Direct bridge from geospatial systems into agent ecosystems.
  • A meaningful differentiation point versus legacy GIS stacks that barely acknowledge agent access.
