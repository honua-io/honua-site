Compatibility

Your existing GIS clients have documented compatibility paths.

ArcGIS Pro, QGIS, OpenLayers, Leaflet, Mapbox GL, ESRI JS API, and any OGC-compliant client can connect to Honua through the standards they already speak where parity is documented. The conformance surface, test counts, and open protocol work are published below with caveats.

What stays connected

Point existing tools at documented surfaces.

For the surfaces on this page, compatibility is scoped by the linked evidence and partial-parity notes.

Desktop

ArcGIS Pro · QGIS

Via GeoServices REST, WFS, WMS, WMTS, and OGC API. Add a service URL and go.

Web maps

OpenLayers · Leaflet · Mapbox GL

Via WMS, WMTS, OGC API Tiles, and Mapbox Vector Tiles. Point the layer at Honua.

Esri ecosystem

ArcGIS Online · Enterprise · JS API

Via GeoServices REST. Honua appears as a standard feature or map service.

API surface

Standards, compatibility, and modern APIs — one deployment.

The documented surfaces share the same binary. A WFS client and a gRPC client can point at the same service where the target surface is enabled.

Surface What it covers When to reach for it
Standards APIs OGC API Features, Tiles, Maps, Records, Processes, Coverages; WFS, WMS, WMTS, WCS; STAC, OData, MVT Existing desktop, web map, and partner clients.
Compatibility APIs GeoServices REST, raster services, geometry services Systems built against ArcGIS Server or GeoServer.
Modern APIs gRPC, SDKs (JS / Python / .NET / MAUI), MCP New apps, automation, Beta field collection, and Preview AI agents.
Pick the surface that fits the job. The same data, auth, and operations back all three.

Open protocol work

The gRPC and MCP specs are public.

The typed spatial APIs behind the runtime are going through the OGC process. Whether you run Honua or not, you can use the same wire format in your own services.

geospatial-grpc

Protobuf schemas for feature, map, tile, and form services.

Published at honua-io/geospatial-grpc. Generate a client in any language with protoc.

geospatial-mcp

An open MCP schema for geospatial agent workflows.

Published at honua-io/geospatial-mcp. Gives Claude, GPT, Cursor, or your own agent a common way to work with spatial data safely.
