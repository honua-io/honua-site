SDKs

The developer story needs first-class pages.

Honua should look like a platform developers can actually build on: typed clients, migration tooling, agent access, compatibility guidance, and direct install paths.

Primary SDKs

Three official SDK lanes, each with a different adoption job.

JavaScript

Apps, migration, compat layers

The JavaScript SDK covers Honua-first clients, OGC usage, Esri compatibility helpers, migration scanning, and runtime parity matrices.

npm install @honua/sdk-js

.NET

Enterprise, admin, and MAUI adjacency

The .NET SDK focuses on gRPC queries, admin workflows, and geocoding, with a natural path into MAUI and broader Microsoft stacks.

dotnet add package Honua.Sdk.Grpc --prerelease
dotnet add package Honua.Sdk.Admin --prerelease

Python

Analysis, automation, and data workflows

The Python SDK gives data teams feature access, geocoding, admin automation, and optional gRPC support for streaming workflows.

pip install honua-sdk
pip install honua-sdk[grpc]

Beyond Basic SDKs

The public developer story is broader than package installs.

Migration toolkit

The JS lane includes scanners, codemods, compat wrappers, and reconciliation helpers for organizations replacing ArcGIS-centric applications.

MCP server package

MCP needs to sit beside the SDK story because agents are now another client class. It is not just a hidden bullet in a docs appendix.

Versioning and compatibility

Server and SDK compatibility contracts are part of the product trust layer. The site should surface those expectations instead of assuming developers will infer them.

Quickstarts

The docs surface should let a team install one SDK, make one authenticated call, and understand where to go next without hunting across repos.

How To Choose

Match the SDK to the adoption motion.

If your team is... Start here Why
Replacing ArcGIS-heavy web apps JavaScript SDK Compat wrappers and migration tooling lower the cost of translation.
Building enterprise services or MAUI apps .NET SDK gRPC and admin clients align well with Microsoft and mobile-adjacent stacks.
Automating data workflows or analysis Python SDK Simple feature access and optional gRPC support fit notebooks, scripts, and pipelines.
Wiring AI assistants or agents into data discovery MCP server Discovery, schema inspection, and filtered queries become tool-safe and explainable.

Field Workflows

The next page after SDKs should be mobile, not a dead end.

If the April platform includes field collection, offline sync, and MAUI-first mobile experiences, the site has to say that directly.
