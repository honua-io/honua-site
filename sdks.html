.NET

Service integrations, gRPC-heavy workloads, and field applications.

The .NET SDK fits backend services, admin automation, and the MAUI-based field toolkit, with a natural path into gRPC application traffic.

Migration and Admin Work

Use the SDK layer to reduce repeated migration and operations work.

Migration Toolkit

Scan, rewrite, and validate common ArcGIS usage patterns.

The JavaScript tooling helps teams locate legacy client usage, use migration automation where it is safe, and reduce the amount of repetitive application migration work.

Admin Clients

Publish services and manage runtime configuration from code.

Admin clients give platform and GIS teams a way to script publication, service management, styles, and manifests in the language they already use.

Specialized Packages

Use the same platform for field work and AI-assisted workflows.

Field Toolkit

Offline sync, forms, and device-friendly workflows.

The MAUI-based field toolkit gives teams offline maps, sync, forms, and operational mobility without adding another product line.

View field toolkit

AI Agents

Connect AI tooling to governed spatial actions.

Use the AI agent package when you want workflows to discover collections, inspect schemas, and call spatial tools through the Model Context Protocol (MCP).
