SDKs

Four SDKs. One client surface.

JavaScript, Python, .NET, and MAUI mobile. Typed contracts everywhere. gRPC under the hood. Same auth, same data model, same call shape — whether you're building a web app, a backend service, a desktop client, or a field tool.

The four SDKs

Pick the language your team already ships in.

JavaScript · web / node

Web apps, services, and migration-heavy frontends.

import { HonuaClient } from "@honua/sdk";

const c = new HonuaClient("https://gis.example.com");
const parcels = await c.features("parcels").query({
  where: "status = 'active'",
  bbox: [-122.42, 37.77, -122.40, 37.79],
});

Python · analysis / pipelines

Analysis, automation, and data workflows.

from honua import Client

with Client("https://gis.example.com") as c:
    parcels = c.features("parcels").query(
        where="status = 'active'",
        bbox=(-122.42, 37.77, -122.40, 37.79),
    )
honua-io/honua-sdk-python →

.NET · services / desktop

Backend services, admin automation, and desktop clients.

using Honua;

await using var c = new HonuaClient("https://gis.example.com");
var parcels = await c.Features("parcels")
    .Where("status = 'active'")
    .InBbox(-122.42, 37.77, -122.40, 37.79)
    .ToListAsync();

Beyond the base client

What else ships with the SDKs.

Each SDK includes base client bindings plus specialized packages for the work that comes up again and again.

Migration helpers · JavaScript

Rewrite ArcGIS client code to the Honua SDK.

Scan JS codebases, identify common ArcGIS client patterns, and rewrite them where the automation can do it safely. Leaves the non-trivial ones for review.

Admin clients · all languages

Publish services, styles, and permissions from code.

The same manifests you commit to Git are also callable as typed operations. Useful when publication is part of a pipeline or a test fixture.
