// the safety architecture

Guardrails are the architecture, not a setting.

None of these are toggles a model can flip mid-conversation. The gates live in the tool handlers and the engine, fixed at registration time by whoever configures the server — an MCP client cannot escalate them.

01 · Plans before actions
Plan mode and PR-first are the enforced defaults in the tool handlers. Planning tools emit plans and evidence bundles — they never apply manifests, submit, or roll back deploy operations on their own.
02 · Changes arrive as GitOps proposals
Agent-initiated changes are recorded as GitOps proposals that never submit immediately. Your team reviews and merges — the agent drafts, it doesn't decide.
03 · Audit by default
Every agent tool call emits an audit record — same schema as a human operator session, with argument redaction built in. The journal is there before you ask for it.
04 · AI cannot edit your data
An engine-level architecture decision, not a config flag. Agents never write features directly; data and config changes go through validated specs with dry runs before anything applies.
05 · Scoped credentials
Agents authenticate with scoped API keys — never the admin password. That hygiene is the documented default, and key scope bounds what any conversation can reach.
06 · Your models, your GPU
Run the agent loop against local models — vLLM or Ollama on hardware you control. No data leaves your network to get the assistance.

// the honest roadmap note

Where the guardrail ladder goes next.

Today every edition gets the same safe defaults: plan mode, PR-first, an audit record on every call. The edition ladder ahead — Pro adds validated agentic GitOps with automated rollback; Enterprise adds human approval workflows, policy controls, and immutable audit — is decided but still being wired into license enforcement.
