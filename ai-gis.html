mcp://honua.example.govanalysis · english → spatial query● EXAMPLE
USER
How many parcels in the SFHA flood zone, and what's the total acreage?
AGENT · tool_call
honua.spatial.query({
  service: "Parcels",
  intersects: "FloodZones/SFHA",
  aggregate: ["count", "sum(area_acres)"]
})
HONUA · result
{
  count: 1284,
  sum_area_acres: 518.4,
  cited_services: ["Parcels@v8", "FloodZones@v3"]
}
AGENT
1,284 parcels intersect the SFHA flood zone — about 518.4 acres in total. Sources: Parcels v8, FloodZones v3. Want me to draft notification letters or render a map?
// the gis workflow

AI across the whole workflow.

Three jobs that used to need a specialist desktop tool, a developer, or an ops team — each one now AI-assisted on the same governed platform.

01 · GIS analysis Spatial analysis from a sentence.

Analysts ask questions in plain English; Honua's MCP tools turn them into typed spatial workflows — intersects, buffers, network analysis, aggregations — with cited source layers and reproducible query records on every answer. The transcript above is this mode running through an MCP-connected agent.

Cited services + provenancealways
Reproducible query recordsaudit-logged
02 · Map & report making Maps, dashboards, reports — prompted.

Describe the map you need; Honua composes the layers, styles, and layout against your real services. Same for dashboards and PDF / HTML reports — produced from typed services, with the exact source service versions baked into every export so the output is auditable, not just pretty.

Cartographers · app teamsvia SDK + MCP
Layouts, dashboards, reportstyped inputs
Source versions in every exportcited
03 · DevOps Honua operates Honua.

honua-devops is the operations agent that watches your Honua deployment. It triages alerts, files Terraform PRs against your IaC repo, runs migration tools, and walks the GitOps loop — governed by your OIDC roles, never bypassing them.
