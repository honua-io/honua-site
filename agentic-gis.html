AI Agents

Let AI workflows work with spatial context directly.

Honua exposes discovery, schema inspection, filtered queries, statistics where supported, and bounded spatial tools through the Model Context Protocol (MCP). Claude, GPT, Cursor, or your own agent discovers what exists, inspects the schema, and runs typed queries in Preview without scraping or hand-written wrappers.

What it gets you

Agents discover the surface at runtime. Schemas are typed. Queries return structured data. No prompt engineering to describe your own REST endpoints.

What the MCP surface covers

Three Preview capabilities agents need to work with spatial data.

Discovery

List collections, layers, and service capabilities.

Agents find what exists without being told. No hidden assumptions baked into the prompt.

Schema

Inspect fields, geometry types, and constraints.

A parcel, a utility line, an address point, and an imagery tile are different shapes. The agent sees that before it writes a query.

Operations

Run filtered queries and bounded spatial predicates.

Preview tools cover bounded operations such as buffer, overlap, contains, crosses, and within where the underlying service enables them.

MCP vs. REST scraping

Agents work better against a typed contract than an undocumented API.

REST endpoints can serve data, but they leave the agent to figure out what exists, what each field means, and which operations are safe. MCP bakes that into the protocol.

Without MCP

Agents guess at endpoints, fields, and geometry behavior.

Prompts hard-code what the API looks like today. Datasets shift. Environments diverge. Silent failures show up in the output before anyone notices.

With MCP

Agents discover, inspect, then act.

Tool calls are typed where the schema covers them. Schemas are introspectable, and supported spatial operations have defined contracts. The agent can adapt when the data changes.

Where teams start

Three practical entry points.

Support & search

Map-backed context for operational questions.

Your assistant inspects service health, lists layers, and retrieves feature context when a user asks about a place or an asset.

Planning

Spatial rule checks before work moves forward.

Buffer, overlap, and collection queries for permit workflows, site reviews, and compliance checks — run inside the agent conversation.

Automation

LLM workflows that touch real spatial state.

Route tasks from internal copilots into governed spatial tools without exposing raw APIs to each workflow author.

Common analyst discovery, query, and bounded spatial tasks can be evaluated over MCP in Preview.
