Cloud-Native Geospatial Platform

A cloud-native GIS platform for migration, apps, mobile, and agents.

Honua is a cloud-native geospatial platform and GIS runtime. It keeps ArcGIS-style and standards-based clients working, adds gRPC for app and mobile performance, and adds MCP for agent-native discovery and query workflows. One operational model, no per-user fees or protocol taxes.

What Honua Is

Keep current GIS clients. Build faster apps. Add agent access.

Compatibility keeps migrations low-risk. gRPC improves application and mobile performance. MCP gives coding agents and assistants a supported way to work with geospatial data.

Why teams switch

  • Replace brittle legacy GIS stacks without breaking existing clients.
  • Adopt open standards and typed SDKs while preserving ArcGIS and OGC entry points.
  • Move app, mobile, and agent workflows onto gRPC and MCP instead of keeping everything trapped in legacy REST patterns.

Architecture

How Honua fits together.

Existing GIS clients enter through compatibility APIs. New apps and mobile clients use gRPC. Agents use MCP. All three paths reach the same Honua runtime and admin APIs.

Commercial Model

Pricing and licensing are part of the product.

Honua is designed to avoid the seat-count and protocol-gating model common in legacy GIS stacks.

No seat pricing

Adding analysts, viewers, or field operators does not force another licensing negotiation.

No protocol or SDK add-ons

GeoServices REST, OGC API, OData, SDKs, and mobile tooling are part of the product, not separate upsells.

Pay for depth, not permission

Commercial tiers map to distributed runtime capability, governance, support, and enterprise readiness.

Clear license split

The server runtime is ELv2. Official SDKs, mobile tooling, and public collateral are Apache 2.0.
