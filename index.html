Honua

Modern GIS for the cloud-native era.

Honua is a modern cloud-native open-core geoplatform built to be easy to operate, broadly interoperable, high performance, AI-ready, and easy to adopt.

It helps organizations lower cloud, operations, and licensing costs while increasing the throughput of GIS work.

Teams can migrate incrementally, run efficiently on modern infrastructure, and deliver maps, apps, analysis, and reports faster.

For users, the result is spatial software that is easier to access, quicker to respond, and better aligned with the way they already work.

Modern vs legacy GIS

The shift is operational, not cosmetic.

Decision area Legacy GIS pattern Modern GIS Platform pattern What improves
Deployment and change control Specialized servers, admin-console changes, and release windows managed outside normal software delivery. Containers, serverless, Kubernetes, Git-managed configuration, Terraform, OIDC, and observable releases. GIS changes become reviewable, repeatable, and diagnosable by the same teams that run other production services.
Cost and scale Long-lived capacity and license commitments are sized around peak demand and vendor-specific deployment assumptions. Elastic runtimes, open-core boundaries, managed cloud primitives, and workload-specific scaling keep capacity closer to use. Teams can reduce idle infrastructure, avoid unnecessary lock-in, and scale the workloads that actually need it.
Interoperability and migration Services, clients, data formats, and user workflows are tightly coupled, so replacement projects become high-risk cutovers. Standards endpoints, service importers, database and file compatibility, and side-by-side migration paths reduce switching friction. Organizations can move service by service while keeping existing users and client applications productive.
Delivery ownership Requests queue behind scarce specialists who translate business needs into GIS-server configuration. Developers, analysts, operators, and GIS teams work through APIs, SDKs, admin surfaces, and governed automation. Maps, apps, analysis, and reports ship through a broader delivery system instead of a narrow tooling bottleneck.
Automation and user access Most workflows assume a human operator in a desktop tool or a custom server console. Web, desktop, mobile, and AI-assisted workflows use the same governed services, metadata, and protocol surface. Routine spatial work can be automated without cutting users off from familiar maps, apps, and operational context.
