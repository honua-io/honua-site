Low Switching Cost

Migration is designed to be staged, not heroic.

Honua is not asking teams to rewrite everything at once. The migration path is built around import, compatibility, scanning, codemods, and staged cutover.

Service import

Bring in live Esri REST services and common spatial files so the first step is onboarding data and services, not replatforming every app.

App converters and compat layers

The JavaScript SDK includes usage scanning, compatibility wrappers, and codemod tooling for ArcGIS-heavy web applications.

Staged cutover

Keep current GIS clients alive through GeoServices REST and OGC while new applications, mobile clients, and agents move onto gRPC, SDKs, and MCP.

Migration reporting

Parity matrices, reconcile flows, and migration reports make the remaining manual work visible before teams commit to a switchover.
