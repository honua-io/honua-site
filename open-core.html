License boundary

Source-available core, open surrounding ecosystem, commercial pro features.

The core runtime is source-available under ELv2. SDKs, tools, protocol work, and ecosystem repositories use Apache-2.0 where applicable. Enterprise and pro capabilities are commercial.

Layer License posture Why it matters
Core server ELv2 source-available runtime. Teams can inspect and self-host the core while Honua keeps a sustainable commercial boundary.
SDKs, tools, specs Apache-2.0 where published as open ecosystem components. Developers can build clients, integrations, and automation without a closed client stack.
Enterprise and pro features Commercial. Paid features align to production needs: governance, support, enterprise identity, managed delivery, and scale controls.

ELv2 is source-available, not OSI open source. Honua keeps that license boundary explicit.
